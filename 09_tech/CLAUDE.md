## 3チャネルのデータ統合が最重要課題

```
EC（Shopify）───────────────────┐
B2B（CRM）───────────────────── → 統合データ基盤 → 分析・意思決定
Restaurant-Bar（POS）──────────┘
```

→ チャネルをまたいだ顧客のLTV・行動パターンを把握する設計が必要

## 技術選定の方針
- **スモールスタート:** 最初はSaaS・既製品を組み合わせる
- **データファースト:** 顧客データの所有権（自社保有）を優先
- **多言語・多通貨対応:** SEA展開を見越した設計
