# /agent-memory — 永続メモリ管理

このコマンドは「覚えておいて」「メモして」「保存して」で自動発動する。

## 永続メモリファイルの場所
`memory/MEMORY.md`

## カテゴリ分類

ユーザーの発言内容から自動でカテゴリを判断して保存する:

| カテゴリタグ | 内容 | 例 |
|---|---|---|
| `#決定事項` | 確定した方針・戦略・選択 | 「MYはバリファーストではなくKLファーストにする」 |
| `#パートナー` | 蔵元・ディストリビューター・B2Bクライアント情報 | 「山田商店の担当はXXさん」 |
| `#アイデア` | まだ検討中のアイデア・仮説 | 「TikTokでの日本酒教育コンテンツが伸びそう」 |
| `#リスク` | 注意すべきリスク・課題 | 「インドネシアの輸入関税が200%になる可能性」 |
| `#タスク` | 後でやること（GitHubに上げるほどではない小タスク） | 「XX社に来週フォローアップする」 |
| `#学び` | 気づき・インサイト | 「B2B提案書は日本語版も用意した方が響く」 |
| `#数字` | 重要な数字・KPI | 「SGのEC月次CVRは3.2%（2026年2月）」 |

## 実行手順

### Step 1: カテゴリ判断
発言内容から最も適切なカテゴリを自動判断する。

### Step 2: エントリー作成

`memory/MEMORY.md` に以下の形式で追記:

```markdown
### YYYY-MM-DD | #カテゴリ
（内容を1-3文で簡潔に記録）
→ 関連: `02_markets/malaysia/` / Issue #XX
```

### Step 3: 確認

```
📝 メモしました（#カテゴリ）:
「YYYY-MM-DD | (保存した内容)」

memory/MEMORY.md に追記しました。
```

---

## 読み込みコマンド

```
/agent-memory read             — MEMORY.md全体を表示
/agent-memory read #パートナー  — 特定カテゴリだけ抽出
/agent-memory search 「キーワード」 — キーワードで検索
/agent-memory clean            — 3ヶ月以上前の完了済みタスクを整理（要確認）
```

---

## セッション開始時の自動読み込み

Claude Codeセッション開始時、このファイルと並んで `memory/MEMORY.md` を自動参照する。
特に `#タスク` と `#決定事項` のエントリーを優先的に確認し、
「前回のセッションで〇〇という決定がありました」と文脈を引き継ぐ。
